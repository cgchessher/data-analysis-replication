Description of Paper Analyses

Their main finding

This paper was a first-of-its-kind in the bumblebee ecology world, because largely bumblebees have cryptic life cycles which are notoriously difficult to sample. Genetic mark-recapture across key life stages (e.g., from spring nest-founding queens to workers, to new sister queens in the following spring) had never been employed in bumblebees before on this scale, offering huge insight into their lives.

As in the title, Carvell et al. were able to tie survival probability (estimated via maximum likelihood) from one season to the next, including queen overwintering, to higher/more quality floral resources in the landscape. More specifically, they were able to estimate survival probabilities for key bumblebee life history stages, which are notoriously cryptic, and therefore difficult to measure in the wild. They also included detailed habitat surveys across the sampled landscape, which allowed them to assess the importance of habitat traits on survival. For instance, percent cover of seminatural vegetation in 1000 radius from the estimated colony location was the best single predictor of lineage survival probability.

Analysis in original paper

Carvell et al. created a modification of a standard Cormack-Jolly-Seber (CJS) genetic mark-recapture model to study patterns of survival over time in colonial species, like bumblebees. They used this analysis to measure the apparent survival of bumblebee lineages (not recapturing individuals) between years, which is to say, between generations/colony cycles, as bumblebees are an annual species. From this, they were able to tie this ‘lineage survival’ probability, which was parameterized by their data using their CJS model, to floral resources in the landscape.

Carvell et al. provide detailed code for their CJS model, which outputs overwintering survival probability, i.e. , from workers in year 1 to spring queens in year 2 from the same lineage, \(\text(\phi_{2i})\)

as a global parameter, where i represents a given family lineage. This parameter represents an estimate of the population-level survival rate for all colonies, which is a shared likelihood calculated with their maximum likelihood model. The likelihood that each lineage contributes will be important for the main replication, the estimation of regression coefficients (see next subsection).

I will be using their provided code to repeat the parameter and model fitting process, so that I can fit:

\[ \text{logit}(\phi_{2i}) = \beta_0 + \beta_1 X_i. \]

The primary information from the data table I will be using is the row-wise vector (Q1, W1, Q2), which represents the number of detected queens in year 1, workers in year 1, and queens in year 2 from a particular family lineage.

My replication analysis

I will be replicating main regression analysis they did to look at the relationships between habitat/land-use variables and per-colony survival probability, shown in Figure 2 (a-d) in the paper. They only did this analysis for colonies/lineages which had a good estimator of approximate colony location (meaning that they caught >1 worker from that colony). They have also provided habitat data per-row for each lineage, meaning that they have used the approximated colony location and extracted landscape-level habitat variables from their associated LIDAR dataset and created variables based on that (e.g., seminatural habitat in 1000 m radius of the approximated colony location).

Parameter and variable definitions

To reiterate, parameters definitions are:

  • Survival probability in a time interval X for a particular family lineage i:

\(\text(\phi_{Xi})\)

  • Estimated Poisson mean counts of workers (x = 1) and queens (x = 2), aka detection:

\(\text(\lambda_{x})\)

  • Recapture probability for a time interval X:

\(\text(P_{X})\)

  • Probability of the colony subsequently remaining undetected, where j is the sampling “round” or “event”. There are 3 samples in the dataset (Q1, W1, Q2). If j = 3, then \(\text(\chi_{3}) = 1\), because no more sampling events remain.

\(\text(\chi_{j})\)

ilt <- function(x) exp(x) / (1 + exp(x))

###
# 3 - Likelihood function code from Carvell papeir for the full dataset (all lineages)
###
totlike <- function(x, F, N) {

  phi  <- ilt(x[1:2])       # phi1, phi2
  lambda <- exp(x[3:4])     # lambda1, lambda2

  p1 <- 1 - dpois(0, lambda[1])
  p2 <- 1 - dpois(0, lambda[2])

  chi <- rep(1, 3)
  chi[2] <- (1 - phi[2]) + phi[2] * dpois(0, lambda[2])
  chi[1] <- (1 - phi[1]) + phi[1] * dpois(0, lambda[1]) * chi[2]

  logL <- 0

  for (i in 1:nrow(F)) {

    fi <- min(which(F[i,] == 1))
    li <- max(which(F[i,] == 1))

    # queens year 1
    if (fi == 1 && li == 1)
      logL <- logL + log(chi[1])

    # q1 -> w1 
    if (fi == 1 && li == 2)
      logL <- logL + log(phi[1] * dpois(N[i,2], lambda[1]) * chi[2])

    # q1 -> w1 -> q2
    if (fi == 1 && li == 3)
      logL <- logL + log(phi[1] * dpois(N[i,2], lambda[1]) *
                         phi[2] * dpois(N[i,3], lambda[2]))

    # workers yr 1
    if (fi == 2 && li == 2)
      logL <- logL + log(tpois(N[i,2], lambda[1]) * chi[2])

    # w1 -> q2
    if (fi == 2 && li == 3)
      logL <- logL + log(tpois(N[i,2], lambda[1]) *
                         phi[2] * dpois(N[i,3], lambda[2]))

    # queens yr 2
    if (fi == 3 && li == 3)
      logL <- logL + log(dpois(N[i,3], lambda[2]))
  }

  return(-logL)  # returning neg log likelihood to maximize
}

Estimating Regression Coefficients

Now, we can move on to the analysis of interest, comparing a baseline null model to a model with a habitat covariate, where we actually estimate the regression coefficients via maximum likelihood (the same model framework we ran above but with beta0 and beta1) as computed by

phi2_i <- plogis(beta0 + beta1 * X)

apr 14

N <- as.matrix(d_sub[, c("Q1", "W1", "Q2")])
F <- (N > 0) * 1

X <- d_sub$'1000.Mixed.Vegetation'

Modified global likelihood model from above. I modified because whereas before we were estimating all relevant parameters for all detected lineages regardless of raw capture rates, here we are biasing our data because we are only interested in lineages for which W1 > 1. Again, we are limiting the data in this way because colony location can only be estimated for lineages where W1 > 1. So, we cannot accurately estimate all parameters. For example, we cannot estimate phi(1) (lineage survival from Q1 –> W1) because all workers in this subset of the data survived to produce workers. Phi(2) is therefore the only survival parameter left to estimate.

#baseline likelihood

loglik_base <- function(par, F, N) {

  phi2 <- ilt(par[1])  # other model estimates both phi1,2, but here phi1 removed.

  lambda1 <- exp(par[2])
  lambda2 <- exp(par[3]) 

  chi3 <- 1 #by default as per supplement

  chi2 <- (1 - phi2) + phi2 * exp(-lambda2)
  # Chi1 is gone because phi1 is gone

  logL <- 0

  for (i in 1:nrow(F)) {

    fi <- min(which(F[i,] == 1))
    li <- max(which(F[i,] == 1))

    if (fi == 2 && li == 2)
      logL <- logL + log(tpois(N[i,2], lambda1) * chi2) #truncated Poisson 

    if (fi == 2 && li == 3)
      logL <- logL + log(tpois(N[i,2], lambda1) *
                         phi2 *
                         dpois(N[i,3], lambda2)) #phi1 removed here

    if (fi == 3 && li == 3)
      logL <- logL + log(dpois(N[i,3], lambda2))
      # phi1 and chi1 removed 
  }

  return(-logL)
}
# habitat regression likelihood
loglik_cov <- function(par, F, N, X) {

  beta0 <- par[1]
  beta1 <- par[2]

  phi2_i <- plogis(beta0 + beta1 * X) #phi2 depends on hab covariate X, also per lineage
  

  lambda1 <- exp(par[3])
  lambda2 <- exp(par[4]) #same as before

  chi3 <- 1

  chi2_i <- (1 - phi2_i) + phi2_i * exp(-lambda2)
  # chi2 now varies by colony because phi is estimated the same, no chi1 bc w1>1

  logL <- 0

  for (i in 1:nrow(F)) {

    fi <- min(which(F[i,] == 1))
    li <- max(which(F[i,] == 1))

    if (fi == 2 && li == 2)
      logL <- logL + log(tpois(N[i,2], lambda1) * chi2_i[i])#trunc poisson as before

    if (fi == 2 && li == 3)
      logL <- logL + log(tpois(N[i,2], lambda1) *
                         phi2_i[i] *
                         dpois(N[i,3], lambda2))

    if (fi == 3 && li == 3)
      logL <- logL + log(dpois(N[i,3], lambda2))
  }

  return(-logL)
}
